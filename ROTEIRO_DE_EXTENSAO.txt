    1. DIAGNÓSTICO E TEORIZAÇÃO 
        1.1. Identificação das partes interessadas e parceiros 
O projeto tem como parte interessada uma empreendedora do setor de vestuário, responsável pela gestão de uma loja de roupas de pequeno porte. O público envolvido inclui a gestora da loja e os acadêmicos responsáveis pela análise dos dados.
O perfil dos participantes inclui:
    • Socioeconômico: pequeno empreendedor do varejo
    • Escolaridade: ensino médio completo
    • Gênero: não especificado
    • Faixa etária: entre 25 e 50 anos
    • Participantes estimados: 1 a 3 pessoas
A colaboração foi realizada de forma direta, com fornecimento de dados reais de vendas para análise, mediante consentimento.

        1.2. Problemática e/ou problemas identificados
Durante conversas com a parte interessada, foi identificado que, embora a loja possua registros de vendas, custos e lucros, esses dados não são analisados de forma estruturada.
A ausência de uma análise detalhada dificulta a identificação de produtos mais lucrativos, controle eficiente de custos e definição de estratégias de venda, o que pode impactar diretamente na rentabilidade do negócio.

        1.3. Justificativa 
O projeto é academicamente relevante por aplicar conceitos de análise de dados em um contexto real de negócio, permitindo transformar dados brutos em informações úteis para a tomada de decisão.
Além disso, o projeto está alinhado com a área de tecnologia e Big Data, utilizando ferramentas como Python para análise e visualização de dados, promovendo aprendizado prático e aplicado.
A motivação do projeto está na necessidade de apoiar pequenas empresas na utilização de dados para melhorar seu desempenho comercial.

        1.4. Objetivos/resultados/efeitos a serem alcançados (em relação ao problema identificado e sob a perspectiva dos públicos envolvidos)
    • Analisar os dados de vendas para identificar padrões de desempenho dos produtos
    • Comparar custos, vendas e lucros para avaliar a eficiência comercial
    • Gerar visualizações gráficas que auxiliem na tomada de decisão

        1.5. Referencial teórico (subsídio teórico para propositura de ações da extensão)
Breve exposição e discussão dos referenciais teóricos utilizados para entender e esclarecer a situação-problema que orienta o projeto, apresentando-as e relacionando-as com o desenvolvimento do projeto. O referencial teórico escolhido deve ser assertivo para justificar as escolhas das ações formuladas, ou seja, obras e autores citados devem apresentar respostas teóricas-científicas apropriadas para os desafios enfrentados durante a execução do projeto de extensão. Aqui no mínimo 3 (três) autores deverão ser referenciados (ver referências bibliográficas da disciplina e outras a critério do professor e ou dos acadêmicos). Sugere-se mínimo de 500 caracteres e máximo de 3 (três) páginas.
	A análise de dados é um elemento fundamental no contexto do Big Data, permitindo extrair informações relevantes a partir de grandes volumes de dados. Segundo Viktor Mayer-Schönberger, o uso de dados possibilita novas formas de tomada de decisão baseadas em evidências.
	De acordo com Thomas H. Davenport, empresas orientadas por dados possuem maior eficiência e competitividade no mercado.
	Além disso, a visualização de dados desempenha um papel importante na interpretação das informações, conforme destacado por Hadley Wickham, facilitando a identificação de padrões e tendências.
	Neste projeto, foram utilizadas técnicas de análise exploratória de dados para compreender o comportamento das vendas e lucratividade dos produtos.
